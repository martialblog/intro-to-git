Kernkonzepte

  • Branch
  • Commit
  • Remote

Branch

Verkettung von Momentaufnahmen des Projekts


                                          E---F---G feature
                                         /
                                A---B---C---D master
  • Die Geschichte aller Änderungen (hinzufügen, ändern, entfernen)
  • Eine Branch ist eine Verkettung von Commits

Commit

Momentaufnahmen des Projekts


                                  commit 980e1ca274a37
                                  Author: Joseph F.
                                  Date: Wed Jun 7

                                  Add cool new feature
  • Besteht aus Daten (der Änderung) und Metadaten (z.B. Autor, Zeitstempel, Notizen)
  • Eine Verkettung von Commits ist eine Branch

Remote

Ein oder mehrere Ablageorte für das Projekt (z.B. GitHub oder GitLab)


                                  (GitLab) A---B---C     origin/master
                                  (lokal)  A---B---C---D master
  • Obligatorisch für gemeinsames Arbeiten
  • Lokale und Remote Branch können synchronisiert werden

Arbeitsbereich anlegen

clone & init

git init

Legt einen neuen leeren Arbeitsbereich an


                                  $ mkdir my-project
                                  $ cd my-project/
                                  $ git init
                                  $ # neuer versteckter Ordner
                                  $ ls -l .git

git clone

Kopiert ein bestehendes Projekt mit HTTP oder SSH


                                  $ git clone https://github.com/martialblog/intro-to-git.git
                                  $ cd intro-to-git/
                                  $ # Die URL wird automatisch eine Remote namens origin
                                  $ grep remote .git/config

Änderungen vornehmen

add, restore & commit

git add

Fügt Änderungen in den Vorbereitungsbereich (Staging Area) hinzu


                                  $ git add my-file.py
  • Metapher: Ein Paket muss zur Post
  • Zuerst packen wir alles in den Karton, ohne das Paket zu schliessen (git add)
  • Danach wird das Paket verschlossen und bekommt einen Stempel (git commit)

git restore

Entfernt Änderungen aus dem Vorbereitungsbereich


                                  $ git restore add-file.py
  • Metapher: Ein Paket muss zur Post
  • Das Paket ist noch nicht verschlossen, also können Dinge noch entfernt werden (git restore)
  • Warum "restore"? Weil wir den letzten bekannten Stand wiederherstellen

git commit

Erstellt einen neuen Commit mit allen Änderungen im Vorbereitungsbereich


                                  $ # -m für einen Titel
                                  $ git commit -m "Add cool new feature"
                                  $ # ohne -m öffnet sich der Editor
                                  $ git commit
  • Metapher: Ein Paket muss zur Post
  • Das Paket wird verschlossen und bekommt einen Stempel (git commit)

Historie betrachten

status, diff & log

git status

Zeigt den aktuellen Status des Arbeitsbereich an


                                  $ git status
                                    Zum Commit vorgemerkte Änderungen:
                                            geändert:       my-app.py

                                    Änderungen, die nicht zum Commit vorgemerkt sind:
                                            gelöscht:       delete-me.py

                                    Unversionierte Dateien:
                                            new-file.py

Fun Fact: Löschen ist für Git auch nur eine Änderung

git diff

Zeigt den Unterschiede im Arbeitsbereich oder Branches an


                                  $ git diff
                                  diff --git a/hello.py b/hello.py
                                  @@ -1 +1 @@
                                  -Hallo Welt
                                  +Hello World

Kann auch anzeigen, was in den nächsten Commit kommt


                                  $ git add goodbye.py
                                  $ git diff --staged
                                  diff --git a/goodbye.py b/goodbye.py
                                  @@ -1 +1 @@
                                  -Tschüss Welt
                                  +Goodbye World

git log

Zeigt die Historie des Projekts an


                                  $ git log
                                  commit c38cee008
                                  Author: Alessandro V.
                                  Date:   Mon Jun 27
                                      Fix bug in login

                                  commit 98ab6ab5f
                                  Merge: b8b0d50 f68bfcf
                                  Author: Joseph F.
                                  Date:   Mon Jun 27
                                      Add login feature
  • Viele tolle Optionen verfügbar:
  • git log --graph
  • git log --oneline --decorate
  • git log --oneline --decorate --graph
  • git log --oneline --grep='Merge'
  • Und viele mehr...

Historie erweitern

branch & merge

git branch

Branches anzeigen, erstellen, löschen


                                  $ git branch
                                  * master
                                  $ git branch neuer-zweig
                                  $ git branch
                                  * master
                                  neuer-zweig
                                  $ git branch -d neuer-zweig
                              

                                  $ git branch new-feature
                                  $ # Alternativ git checkout new-feature
                                  $ git switch new-feature
                                  $ git branch
                                  master
                                  * new-feature

git merge

Entwicklungszweige zusammenführen


                              Vor Merge
                                      A---B---C topic
                                     /
                                D---E---F---G master

                              Nach Merge
                                      A---B---C feature
                                     /         \
                                D---E---F---G---H master

H enthält alle Änderungen von A, B und C


                                  $ git branch new-feature
                                  $ git switch new-feature
                                  $ # Änderung vornehmen, git add & git commit
                                  $ git switch master
                                  $ git merge new-feature

git merge CONFLICT!

Entsteht, wenn auf Branches dieselben Zeilen in einer Datei geändert wurden und man mergen möchte


                              $ git merge new-feature
                              Auto-merging app.py
                              CONFLICT (content): Merge conflict in app.py
                              Automatic merge failed;
                              fix conflicts and then commit the result.
  • git status hilft beim Identifizieren der Dateien
  • Git markiert Fehler innerhalb der Dateien
  • In den Dateien wählt man die gewünschten Zeilen aus, entfernt die anderen
  • Mit git commit finalisiert man den Merge
  • Wenn alle Stricke reissen: git merge --abort

Mit Remotes arbeiten

push & pull

git remote

Remote URLs anzeigen, erstellen, löschen


                                  $ git remote
                                  origin
                                  $ git remote add github git@github.com:martialblog/intro-to-git.git
                                  $ git remote
                                  origin
                                  github
                                  $ git remote remove github

git push

Commits und Dateien zu einer Remote hochladen


                                  $ # git push REMOTE-Name BRANCH-Name
                                  $ git push origin master
                              

                                  # Vor Push
                                  (GitLab) A---B  origin/master
                                  (lokal)  A---B---C---D master

                                  # Nach Push
                                  (GitLab) A---B---C---D origin/master
                                  (lokal)  A---B---C---D master
  • Eine Remote Branch hat den Remote Namen als Prefix
  • Wenn sich die Historie zu sehr unterscheidet, kann nicht gepusht werden
  • Grund: Zentrale Historie würde überschrieben

git pull

Commits und Dateien von einer Remote herunterladen


                                  $ git pull

Fun Fact: git pull ist ein Kürzel für git fetch & git merge. Heißt aber auch, merge conflict ist möglich
